# F3.3 — Fila deliberar (professor)

**Rota:** `/solicitacoes?to=me`  
**Fluxo:** F3  
**Shell:** AppLayout  
**Plataforma:** Web + Mobile  
**Prioridade:** P2  
**Capability:** `request.deliberate`  
**API primária:** `GET /requests?canDeliberate=true`

---

## 1. Propósito

Solicitações para deliberar com ações em lote.

## 2. Layout e estrutura (Auto Layout)

```
PageContent
  ├─ Header row: H1 + Button primary "N/A"
  ├─ FilterBar (gap space/sm): tipo, curso, atraso
  ├─ DataTable / Card list
  │    Colunas: Número, Aluno, Tipo, Prazo, SLA
  └─ Pagination footer
```

## 3. Seções e conteúdo

Tabela com zebra `surface/subtle` em hover row. SLA breach: texto `status/danger`.

## 4. Componentes DS/* obrigatórios

`DS/DataTable`, `DS/Button`, `DS/Badge`, `DS/Input` (busca), `DS/EmptyState`, `DS/Skeleton`

## 5. Estados da interface

Loading skeleton rows | Empty | Error AlertBanner | Filtro sem resultados

## 6. Responsividade

Tabela → cards empilhados em mobile; filtros em Sheet/drawer

## 7. Acessibilidade (WCAG 2.1 AA)

Cabeçalhos `th` scope; ordenação indicada com ícone

## 8. HATEOAS / ações condicionais

Ações por linha via `_links` — ocultar botão se rel ausente

## 9. Instruções Figma

Usar pattern `Pattern/ListPage`. Criar variantes Loaded e Empty.
